2. Objectifs & Description

Problématique

Les étudiants de l'IFRI rencontrent régulièrement des difficultés dans certaines matières, tandis que d'autres étudiants, plus avancés ou maîtrisant ces sujets, pourraient les aider. Il n'existait pas de plateforme dédiée pour faciliter cette mise en relation.

Solution apportée

IFRI MentorLink est une application web qui permet :

  • La création de profils académiques avec compétences, lacunes et disponibilités
  • La publication d'offres (je propose de tutorer) et de demandes (j'ai besoin d'aide)
  • Un algorithme de matching intelligent scoré sur 100 points
  • Une messagerie instantanée intégrée pour organiser les sessions

Technologies utilisées

Python 3.11 Flask 3.x PostgreSQL 15 SQLAlchemy JWT Socket.IO Tailwind CSS pytest

4. Algorithme de matching

Le score total est calculé sur 100 points répartis en 4 critères :

Compatibilité matière
40 pts
Disponibilités communes
30 pts
Proximité filière
15 pts
Proximité niveau
15 pts

Détail des critères

  • Compatibilité matière : 40 pts si la matière principale de l'offre correspond à celle de la demande. Bonus jusqu'à 20 pts supplémentaires si des matières communes existent dans les compétences/lacunes respectives.
  • Disponibilités communes : 10 pts par créneau commun calculé (intersection des plages horaires), plafonné à 30 pts.
  • Proximité filière : 15 pts si même filière, dégressif selon la distance entre filières dans un tableau de correspondance prédéfini.
  • Proximité niveau : 15 pts si le mentor est d'un niveau supérieur d'exactement 1 rang, dégressif au-delà, 0 si le mentor est moins avancé.

Seuil minimum : 30/100 — en dessous, le matching n'est pas présenté.

5. Implémentation technique

Backend — modules clés

  • app/__init__.py — Factory create_app(), enregistrement des blueprints, CORS
  • app/middleware/auth.py — Décorateur @token_required pour vérifier le JWT
  • app/services/matching.py — Algorithme de scoring pur (sans dépendance Flask)
  • app/sockets/__init__.py — Événements Socket.IO (connexion, join_room, new_message)

Frontend — architecture JS

  • js/api.js — Client API centralisé exposé via window.API. Gère le token JWT, les redirections 401, et expose tous les modules (auth, profile, offers, matching, messages, notifications).
  • js/matieres-loader.js — Catalogue des matières par filière exposé via window.MATIERES

Sécurité

  • Mots de passe hashés avec Werkzeug (PBKDF2-SHA256)
  • Authentification par JWT (expiration configurable, défaut 24h)
  • CORS configuré pour n'accepter que l'origine du frontend
  • Vérification de propriété sur toutes les opérations destructives

7. Manuel d'utilisation

1 — Inscription / Connexion

Accéder à /Frontend/pages/signin.html. L'animation atome représente l'IFRI avec les 5 filières en orbite. Créer un compte via l'onglet "Inscription", puis compléter l'onboarding.

2 — Onboarding

En 4 étapes : choisir sa filière et son niveau, sélectionner ses compétences, ses lacunes, puis ses créneaux de disponibilité.

3 — Publier une offre ou demande

Depuis "Offres & Demandes", cliquer sur "+ Créer", choisir le type (offre/demande), la matière, le format, et publier.

4 — Explorer les matchings

La page "Matching" propose un système de cartes swipables. Le score sur 100 est visible avec le détail par critère. Cliquer "Contacter" pour ouvrir une conversation.

5 — Messagerie

La page "Messages" affiche toutes les conversations. Les messages sont envoyés en temps réel via Socket.IO.

6 — Paramètres

Modifier filière, niveau, bio, compétences, lacunes et disponibilités à tout moment depuis "Paramètres".
